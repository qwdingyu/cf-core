## 一、设计理念

**两个 Token 搞定一切**：只需 `CLOUDFLARE_API_TOKEN` + `TURSO_API_TOKEN`，其余全部自动生成。

| 自动生成的凭证 | 用途 | 安全级别 |
|--------------|------|---------|
| ADMIN_TOKEN | 管理后台认证 | 自动生成 48 字符 hex |
| RATE_LIMIT_SALT | IP 哈希盐值 | 自动生成 48 字符 hex |
| JWT_SECRET | JWT 签名密钥 | 自动生成 48 字符 hex |
| CREDENTIALS_ENCRYPTION_KEY | AES-256 加密 | 自动生成 64 字符 hex |
| Turnstile Site Key / Secret | 人机验证 | Cloudflare API 自动创建 |

**凭证持久化**：所有凭证保存在 `.credentials/` 目录（已 gitignore），支持：
- 幂等重跑（已有凭证自动复用）
- 断点续传（已完成的步骤自动跳过）
- 一键推送到 GitHub Actions Secrets

## 二、脚本清单

```
templates/.deploy/
├── shared.sh              # 公共函数库（日志/检查/API封装）
├── one-click.sh           # 一键入口（编排所有步骤）
├── 00-init-project.sh     # Git 初始化 + GitHub 仓库创建
├── 01-setup-turso.sh      # Turso 数据库创建 + 迁移
├── 02-setup-secrets.sh    # 自动生成所有 Secrets
├── 03-deploy-worker.sh    # 构建 + 部署 + 冒烟测试
├── 04-bind-domain.sh      # 域名绑定 + DNS + SSL
├── 05-setup-turnstile.sh  # Turnstile Widget 自动创建
├── 06-setup-github.sh     # GitHub Actions + Secrets + Variables
└── workflows/
    ├── deploy.yml           # CI/CD 部署流水线
    ├── cleanup-schedule.yml # 定时清理（每 2 小时）
    └── backup-daily.yml     # 每日数据库备份
```

## 三、快速开始

### 3.1 前置条件

| 工具 | 安装命令 | 用途 |
|------|---------|------|
| Node.js 22+ | `brew install node` | 运行时 |
| jq | `brew install jq` | JSON 解析 |
| gh | `brew install gh && gh auth login` | GitHub CLI |
| openssl | 系统自带 | Token 生成 |
| curl | 系统自带 | API 调用 |

### 3.2 一键部署

```bash
# 从模板创建新项目
cp -R templates/cf-shop ../my-shop
cd ../my-shop
npm install

# 一键部署（只需两个 Token）
CLOUDFLARE_API_TOKEN="cfat_xxx" \
TURSO_API_TOKEN="turso_xxx" \
bash .deploy/one-click.sh
```

### 3.3 带自定义域名

```bash
CLOUDFLARE_API_TOKEN="cfat_xxx" \
TURSO_API_TOKEN="turso_xxx" \
DOMAIN="shop.eforge.xyz" \
bash .deploy/one-click.sh
```

### 3.4 跳过某些步骤

```bash
# 跳过域名和 Turnstile
SKIP_DOMAIN=true SKIP_TURNSTILE=true \
CLOUDFLARE_API_TOKEN="cfat_xxx" TURSO_API_TOKEN="turso_xxx" \
bash .deploy/one-click.sh

# 只运行特定步骤
ONLY=03 CLOUDFLARE_API_TOKEN="cfat_xxx" TURSO_API_TOKEN="turso_xxx" \
bash .deploy/one-click.sh
```

## 四、各步骤详解

### Step 00: 项目初始化

| 操作 | 工具 |
|------|------|
| 检查前置条件（node/npm/gh/jq/openssl） | `command -v` |
| 安装项目依赖 | `npm ci` |
| Git 初始化 | `git init -b main` |
| 创建 GitHub 私有仓库 | `gh repo create --private` |
| 推送到 GitHub | `git push -u origin main` |

### Step 01: Turso 数据库

| 操作 | API |
|------|-----|
| 获取 Turso 组织名 | `GET /organizations` |
| 创建数据库（幂等） | `POST /organizations/{org}/databases` |
| 创建 Token | `POST /organizations/{org}/databases/{db}/tokens` |
| 执行 SQL 迁移 | libsql Node.js 客户端 |

### Step 02: Secrets 生成

所有凭证自动生成并保存到 `.credentials/`，**幂等**：已有则复用。

### Step 03: Worker 部署

| 操作 | 命令 |
|------|------|
| 构建前端（可选） | `npm run build:frontend` |
| 部署 Worker + Secrets | `wrangler deploy --secrets-file` |
| 等待 Worker 启动 | 轮询 `/api/health` |
| 冒烟测试 | `node scripts/smoke-readonly.mjs` |

### Step 04: 域名绑定

| 操作 | API |
|------|-----|
| 获取 Account ID | `GET /accounts` |
| 推导 Zone ID | `GET /zones?name={zone}` |
| 绑定 Custom Domain | `POST /accounts/{id}/workers/domains` |
| 等待 HTTPS 证书 | 轮询域名（最多 24 次 × 5s） |

### Step 05: Turnstile

| 操作 | API |
|------|-----|
| 创建 Widget | `POST /accounts/{id}/challenges/widgets` |
| 写入 Site Key 到数据库 | libsql 客户端 |
| 设置 Secret 到 Worker | `wrangler secret put` |

### Step 06: GitHub Actions

| 操作 | 工具 |
|------|------|
| 安装 3 个 Workflows | 模板替换 + 复制 |
| 推送所有 Secrets | `gh secret set` |
| 设置 Variables | `gh variable set` |
| 提交并触发首次部署 | `git push` |

## 五、CI/CD 流水线

### deploy.yml — 主部署流水线

```
push to main → checkout → npm ci → type-check → test
  → build frontend → migrate DB → deploy Worker
  → bind domain → smoke test → 部署报告
```

### cleanup-schedule.yml — 定时清理

```
每 2 小时 → POST /api/admin/cleanup（清理过期订单/凭证/Token）
```

### backup-daily.yml — 每日备份

```
每日 03:00 UTC → 导出 Turso 数据库 → 上传 GitHub Artifacts（保留 7 天）
```

## 六、凭证文件结构

```
.credentials/
├── ADMIN_TOKEN              # 管理后台 Token
├── RATE_LIMIT_SALT          # IP 哈希盐值
├── JWT_SECRET               # JWT 签名密钥
├── CREDENTIALS_ENCRYPTION_KEY # AES-256 加密密钥
├── TURSO_URL                # Turso 数据库 URL
├── TURSO_TOKEN              # Turso 认证 Token
├── TURNSTILE_SITE_KEY       # Turnstile 前端 Key
├── TURNSTILE_SECRET_KEY     # Turnstile 后端 Key
├── PROJECT_NAME             # 项目名
├── GITHUB_REPO              # GitHub 仓库
├── WORKER_NAME              # Worker 名称
├── DOMAIN                   # 自定义域名
├── BASE_URL                 # 访问地址
├── DB_NAME                  # 数据库名
└── .deploy-progress         # 步骤进度（断点续传）
```

**安全**：`.credentials/` 已在 `.gitignore` 中排除，不会提交到 Git。

## 七、断点续传

脚本通过 `.credentials/.deploy-progress` 文件跟踪已完成的步骤。重跑时自动跳过。

```bash
# 清除进度，全部重跑
rm .credentials/.deploy-progress
bash .deploy/one-click.sh

# 只重跑 Step 03
ONLY=03 bash .deploy/one-click.sh
```
